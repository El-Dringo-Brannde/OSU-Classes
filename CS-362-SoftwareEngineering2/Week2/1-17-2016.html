Why Testing Matters

  • Disney's Lion King PC game in 1994
    • Developers only tested game on their system, and didn't account for various computer configurations
  • NASA Mars Polar Lander 1999
    • A single bit was probably tripped when the legs came out for landing
    • Therefore the Lander already thought it landed and didn't deploy any thrusters or parachutes
    • It was tested by a team, but they reset the computer to switch the bit back before actually testing it with the legs down
  • Testing vs Static Analysis
    • Static Analysis
      • Examining a programs source code without executing it.
      • Sort of happens when a compiler goes through your program before making executable
      • Could give a false positive by warning about a potential problem that wouldn't show up when running
      • There are static analsis programs to check Java programs
      • Static analysis may be able to find small bugs, but generally checks the software does what it is supposed to do.
  • Testing occurs at every level of the sofware development lifecycle
    • Although exhaustive testing is impossible there multi billion possibilities to test

    Of course a bug is cheaper and easier to fix the sooner it is found and rectified
